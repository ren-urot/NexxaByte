# Hex-Grid Pattern Fix Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the tiled diamond-pattern background (which uses a non-tileable banner-style source asset and renders as a busy, patchy micro-texture) with a single evenly-distributed hex-grid graphic rendered via `background-size: cover`, matching a user-supplied reference image of a clean, spacious hexagon-line grid.

**Architecture:** A new SVG asset (`assets/images/pattern-hexgrid.svg`) is generated by a small Python script that computes an evenly-spaced pointy-top hexagon tessellation with sparse dot accents. `assets/css/parallax.css`'s existing `::before` pattern layer switches from `background-repeat: repeat` (small tile) to `background-repeat: no-repeat; background-size: cover` (one graphic scaled to fill each section), and gains a vertical overdraw margin so scroll-driven drift never exposes empty space. `assets/js/main.js`'s `initParallax()` gets its offset calculation clamped to stay within that overdraw margin. The old `pattern-diamonds.svg` asset is deleted. No other mechanism changes: opacity tiers, `prefers-reduced-motion` guard, and the `--pattern-offset` custom property handoff between CSS and JS all stay exactly as they are today.

**Tech Stack:** Plain HTML5/CSS3/vanilla ES5 JS static site (no build tooling), Python 3 (asset generation only, not shipped — output is a static SVG file), headless Chrome CLI for visual verification (confirmed working in this sandbox; the Playwright MCP screenshot tools are known to hang here and must not be used).

## Global Constraints

- Single shared SVG asset for both dark-hero (`.parallax-hero`) and light-section (`.section-shapes-host`) contexts, differentiated only by CSS `opacity` — no separate color variants (spec: Asset section).
- `assets/images/pattern-diamonds.svg` must be deleted; `assets/images/pattern-hexgrid.svg` is the sole replacement (spec: Asset section, "nothing else references it after this change").
- Opacity tiers stay exactly as currently set: `.parallax-hero::before` and `.section-dark.section-shapes-host::before` at `opacity: 0.35`; plain `.section-shapes-host::before` at `opacity: 0.08` (spec: CSS Changes section — these values are unchanged from the current, already-fixed state).
- Parallax speed multiplier stays `0.1` (`rect.top * 0.1`); only the clamp range (`Math.max(-50, Math.min(50, offset))`) is new (spec: JS Changes section).
- `prefers-reduced-motion: reduce` must continue to fully disable drift — do not touch that guard (spec: What Stays the Same).
- Do not use Playwright MCP screenshot tools (`browser_take_screenshot`, `page.screenshot()`) for any verification in this plan — they hang indefinitely in this sandbox. Use the headless Chrome CLI method instead: `"/Applications/Google Chrome.app/Contents/MacOS/Google Chrome" --headless --disable-gpu --screenshot=<output-path> --window-size=<W>,<H> --hide-scrollbars <url>`, run via a backgroundable shell command (it exits on its own once the screenshot is written — no need to kill it).

- [ ] **Step 1: Write the generation script**

Save this as a scratch file (e.g. `/tmp/gen-hexgrid.py` — anywhere outside the repo is fine, it's not a project dependency):

```python
import math
import random

WIDTH, HEIGHT = 1600, 900
HEX_RADIUS = 100        # center-to-vertex distance
STROKE_COLOR = "#e6e7e8"
STROKE_WIDTH = 1
STROKE_OPACITY = 0.25
DOT_RADIUS = 3
DOT_FILL_OPACITY = 0.4
DOT_FRACTION = 0.18      # fraction of vertices that get a dot accent
SEED = 42

def hex_corners(cx, cy, r):
    # Pointy-top hexagon: first vertex points straight up.
    corners = []
    for i in range(6):
        angle_deg = 60 * i - 30
        angle_rad = math.radians(angle_deg)
        x = cx + r * math.cos(angle_rad)
        y = cy + r * math.sin(angle_rad)
        corners.append((round(x, 2), round(y, 2)))
    return corners

def generate():
    random.seed(SEED)
    hex_w = math.sqrt(3) * HEX_RADIUS   # horizontal spacing between centers in a row
    hex_h = 1.5 * HEX_RADIUS            # vertical spacing between rows

    polygons = []
    dot_candidates = []
    seen_vertices = set()

    row = 0
    y = -HEX_RADIUS
    while y - HEX_RADIUS < HEIGHT + hex_h:
        x_offset = (hex_w / 2) if (row % 2 == 1) else 0
        col = 0
        x = -hex_w + x_offset
        while x - HEX_RADIUS < WIDTH + hex_w:
            corners = hex_corners(x, y, HEX_RADIUS)
            points_str = " ".join("{},{}".format(px, py) for px, py in corners)
            polygons.append(points_str)
            for px, py in corners:
                key = (round(px, 1), round(py, 1))
                if key not in seen_vertices:
                    seen_vertices.add(key)
                    dot_candidates.append((px, py))
            x += hex_w
            col += 1
        y += hex_h
        row += 1

    dot_candidates.sort()
    num_dots = int(len(dot_candidates) * DOT_FRACTION)
    dots = random.sample(dot_candidates, num_dots)

    lines = []
    lines.append('<?xml version="1.0" encoding="UTF-8"?>')
    lines.append(
        '<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 {} {}">'.format(WIDTH, HEIGHT)
    )
    lines.append(
        '  <g fill="none" stroke="{}" stroke-width="{}" stroke-opacity="{}">'.format(
            STROKE_COLOR, STROKE_WIDTH, STROKE_OPACITY
        )
    )
    for points_str in polygons:
        lines.append('    <polygon points="{}"/>'.format(points_str))
    lines.append('  </g>')
    lines.append(
        '  <g fill="{}" fill-opacity="{}">'.format(STROKE_COLOR, DOT_FILL_OPACITY)
    )
    for px, py in dots:
        lines.append('    <circle cx="{}" cy="{}" r="{}"/>'.format(px, py, DOT_RADIUS))
    lines.append('  </g>')
    lines.append('</svg>')
    return "\n".join(lines)

if __name__ == "__main__":
    print(generate())
```

### Task 3: Clamp the parallax offset to the overdraw margin

**Files:**
- Modify: `assets/js/main.js`

**Interfaces:**
- Consumes: the 60px overdraw margin established in Task 2 (`top: -60px; bottom: -60px`).
- Produces: `--pattern-offset` values clamped to `[-50, 50]` (px), read by the `transform: translateY(var(--pattern-offset, 0))` rule from Task 2.

- [ ] **Step 1: Locate the current `update()` function inside `initParallax()`**

Run: `grep -n "function update\|pattern-offset\|rect.top" /Users/ren/Desktop/NexxaByte/assets/js/main.js`

You should find a block inside `initParallax()` that looks like:

```js
function update() {
  for (var i = 0; i < hosts.length; i++) {
    var rect = hosts[i].getBoundingClientRect();
    hosts[i].style.setProperty('--pattern-offset', (rect.top * 0.1) + 'px');

```

- [ ] **Step 2: Add the clamp**

Replace that function body with:

```js
function update() {
  for (var i = 0; i < hosts.length; i++) {
    var rect = hosts[i].getBoundingClientRect();
    var offset = rect.top * 0.1;
    offset = Math.max(-50, Math.min(50, offset));
    hosts[i].style.setProperty('--pattern-offset', offset + 'px');
  }
  ticking = false;
}
```

Do not change anything else in `initParallax()` — the `prefers-reduced-motion` guard, the `ticking` rAF-throttle flag, the `requestUpdate()` function, and the `scroll`/`resize` listener registrations all stay exactly as they are.
